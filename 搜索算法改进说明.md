# 搜索算法改进说明

## 概述

本次改进将 deino-lit-query 项目的搜索算法升级为 **v4 投票计数 + 唯一性核对** 机制。两个版本（前端 `index.html` 和后端 `app.py`）保持逻辑完全一致。

**核心变更**：将查询拆分为单词后，每个单词独立在全库检索标题匹配，通过投票计数和逐轮唯一性核对来确定最相关的文献。

---

## 算法核心逻辑（v4）

### 1. 单词分割

查询字符串按空格/空白字符分割为单词数组，保留单词内部的标点符号。

| 输入示例 | 分割结果 |
|---------|---------|
| `"deino radiodurans radiation resistance"` | `["deino", "radiodurans", "radiation", "resistance"]` |
| `"radiodurans."` | `["radiodurans."]` |
| `"DNA repair"` | `["DNA", "repair"]` |

### 2. 投票计数 + 唯一性核对

**核心流程：**

1. **每个单词独立在全库检索**标题匹配（不递进过滤，而是独立投票）
2. 维护投票计数器：`Map<number, {entry, count}>`
3. 先处理**前8个单词**（或全部单词，取最小值），统计投票
4. **核对**：最高票数的文献是否**唯一**
   - **唯一**（只有一篇文献票数最高）→ 直接返回该文献
   - **不唯一**（多篇文献并列最高票）→ 继续下一步
5. **增加第9个单词**（如果存在），将其匹配结果加入投票池，重新统计
6. 再次核对最高票是否唯一 → 唯一则返回，否则继续
7. 以此类推，直到用完所有单词或得到唯一解

**示例：**

输入：`"deino radiodurans radiation resistance dna repair protein structure function"`

| 单词 | 匹配文献 | 投票变化 |
|------|---------|---------|
| `"deino"` | {A, B, C, D} | A=1, B=1, C=1, D=1 |
| `"radiodurans"` | {A, B, E} | A=2, B=2, C=1, D=1, E=1 |
| ...处理完8个单词 | ... | A=8, B=7, C=5... |
| **核对** | A是唯一最高(8票) | **→ 返回A** |

**并列最高时的处理：**

如果8个单词后：A=6票，B=6票（并列最高）→ 不唯一 → 增加第9个单词

- 单词9 `"function"` → 匹配到 {A, C} → A=7票，C增加票数
- 再次核对：A=7票，B=6票 → A是唯一最高 → **返回A**

### 3. 边界处理

| 场景 | 处理方式 |
|------|---------|
| 查询少于8个单词 | 直接用全部单词统计投票 |
| 某个单词无匹配 | 不影响投票（该单词不投任何票） |
| 所有单词用完仍不唯一 | 按票数降序排列，返回前30条 |
| 完全没有匹配的单词 | 进入 fallback 链 |

## 保留的原有功能

| 功能 | 触发条件 | 优先级 |
|------|---------|--------|
| **序号精确匹配** | 输入纯数字（如 `189`） | 最高 |
| **标题搜索** | 输入非纯数字时先进行标题投票计数 | 主流程 |
| **作者名搜索** | 标题无结果时搜索作者字段 | Fallback |
| **PMID 精确匹配** | 作者搜索无结果时 | Fallback |
| **摘要全文搜索** | 标题和作者均无结果时 | 最后 Fallback |

## 前后端逻辑一致性说明

两个版本的搜索逻辑完全相同：

1. 都先检查精确序号匹配
2. 都执行投票计数 + 唯一性核对算法
3. 都先处理前8个单词，检查唯一性
4. 不唯一时逐轮增加单词，直到唯一或用完
5. 所有单词用完仍不唯一 → 按票数排序，取前 30 条
6. 都有相同的 Fallback 搜索链（作者 → PMID → 摘要）

## 版本演进

| 版本 | 匹配策略 | 特点 |
|------|---------|------|
| v1 | 简单包含匹配 | 直接检查标题是否包含查询字符串 |
| v2 | 字符递进匹配 | 按字符逐层递进过滤（如 d → de → dei → dein → deino） |
| v3 | 渐进式单词递进 + 相似度评分 | 按单词逐层递进过滤，配合相似度评分排序 |
| **v4（当前）** | **投票计数 + 唯一性核对** | **每个单词独立投票，通过唯一性核对确定最相关文献** |
