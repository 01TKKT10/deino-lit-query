# Deinococcus radiodurans 文献查询系统 — 项目架构说明

## 一、项目概述

本项目是一个基于纯前端的学术文献检索 Web 应用，用于查询和管理《耐辐射奇球菌研究》专著所需的 1,344 篇 PubMed 文献。

**核心特征：**
- 无需后端服务器即可运行（双击 `index.html` 或 `python main.py` 一键启动）
- 全部数据来自 PubMed API 的标准化字段
- 支持序号/关键词/作者/PMID 搜索
- 文献详情含标题、作者、期刊、年份、PMID、DOI、PMCID、摘要、MeSH Terms

## 三、各文件详细说明

### 3.1 index.html（核心文件）

**作用**：项目的唯一入口。包含完整的 HTML 结构、CSS 样式和 JavaScript 逻辑。

**依赖**：
- 同级目录下的 `database.json`（通过 `fetch('database.json')` 加载）
- 无其他外部依赖（不依赖 CDN、不依赖后端 API）

**注意**：
- 由于浏览器安全策略（CORS），直接双击打开 `index.html` 会导致 `database.json` 加载失败
- 必须通过 `python main.py` 启动本地 HTTP 服务器后访问
- 或在任何静态文件服务器（Nginx/Apache/GitHub Pages 等）上部署

### 3.2 database.json（数据文件）

**作用**：1,344 篇文献的完整数据库。

**数据来源**：
1. 从 CSV 文件提取全部 PMID（`csv-Deinococcu-set.csv`，1,344 行）
2. 通过 PubMed E-utilities API 的 `efetch.fcgi` 接口批量获取每篇文献的标准 XML
3. 解析 XML 提取以下标准化字段：
   - `pmid` — PubMed ID
   - `title` — 文章标题
   - `authors` — 作者列表（逗号分隔）
   - `journal` — 期刊全称
   - `year` — 发表年份
   - `doi` — DOI
   - `pmcid` — PubMed Central ID
   - `abstract` — 摘要全文
   - `mesh_terms` — MeSH 主题词列表
   - `pub_types` — 出版物类型列表

**数据格式示例**：
```json
{
  "1": {
    "number": 1,
    "pmid": "41148371",
    "title": "Cell Division and NlpC/P60 Endopeptidases...",
    "authors": "Kamara M, Ishiwa A, Narumi I",
    "journal": "Biosci Biotechnol Biochem",
    "year": 2026,
    "doi": "10.1093/bbb/zbad190",
    "pmcid": null,
    "abstract": "Deinococcus radiodurans shows ...",
    "mesh_terms": ["Cell Division", "Endopeptidases", "Peptidoglycan"],
    "pub_types": ["Journal Article"]
  }
}
```

**文件大小**：约 2.5 MB（JSON 格式，含换行和缩进）

### 3.3 main.py（推荐启动器）

**作用**：一键启动本地 HTTP 服务器并自动打开浏览器。

**原理**：使用 Python 标准库 `http.server` 和 `socketserver`，无需安装任何第三方包。

**启动流程**：
1. 检查 `index.html` 和 `database.json` 是否存在
2. 在端口 8765 启动 HTTP 服务器
3. 延迟 1 秒后自动调用系统默认浏览器打开 `http://localhost:8765`
4. 静默日志（不显示每个 HTTP 请求）
5. 按 `Ctrl+C` 停止服务器

**依赖**：仅需 Python 3（无 pip install 需求）

### 3.4 app.py（Flask 后端版本，可选）

**作用**：Flask Web 应用，提供后端 API 路由。

**路由**：
- `GET /` — 渲染首页
- `GET /api/search?q=关键词` — 搜索 API，返回 JSON
- `GET /api/entry/<序号>` — 获取单篇文献详情

**与 main.py 的区别**：
- main.py 是纯静态文件服务器（前端处理所有搜索逻辑）
- app.py 是前后端分离（后端处理搜索，前端只负责渲染）

**依赖**：需安装 Flask（`pip install flask`）

### 3.5 templates/index.html & static/style.css

**作用**：Flask 版本使用的模板文件和样式表。

**注意**：当使用 `main.py` 启动时，这些文件**不被使用**。
`main.py` 直接读取项目根目录下的 `index.html`（已将 CSS 和 JS 内联）。

## 五、技术架构图

```
用户浏览器
    │
    ├─→ 输入查询（序号/关键词/作者/PMID）
    │
    ├─→ main.py (Python http.server)
    │       │
    │       ├─→ 返回 index.html（含 CSS + JS）
    │       └─→ 返回 database.json（文献数据）
    │
    └─→ 前端 JS 处理搜索逻辑
            │
            ├─→ fetch('database.json') 加载全部数据到内存
            ├─→ 在内存中执行字符串匹配搜索
            └─→ 渲染结果卡片 + 模态框详情
```

**关键点**：
- 所有搜索逻辑在前端完成（纯客户端）
- `database.json` 一次性加载后缓存在内存中
- 无后端数据库，无 API 调用延迟
- 适合本地离线使用

## 六、数据来源说明

### 6.1 原始数据文件（用户提供）

| 文件 | 内容 | 行数 |
|------|------|------|
| `csv-Deinococcu-set.csv` | 文献元数据（PMID/标题/作者/期刊/年份） | 1,344 行 |
| `abstract-Deinococcu-set.txt` | 文献摘要全文（含标题/作者/摘要/PMID） | 1,344 篇 |
| `目录K1.md` | 专著12章目录框架 | 约 200 个知识点 |

### 6.2 数据处理流程

```
csv-Deinococcu-set.csv
    │
    ├─→ 提取全部 PMID（1,344 个）
    │
    └─→ PubMed E-utilities API (efetch.fcgi)
            │
            └─→ 返回标准 XML（含 title/authors/journal/
                abstract/DOI/PMCID/MeshTerms 等）
                    │
                    └─→ 解析 XML → 构建规范化 JSON
                        （database.json）
```

**为什么要从 PubMed API 重新获取，而不是直接用 txt 文件？**

原始 `abstract-Deinococcu-set.txt` 的格式不统一：
- 部分文献缺少 `Author information:` 行
- 日期行位置不固定（有的有 "Epub" 行，有的没有）
- 俄文/法文/中文文献的格式特殊
- 摘要和标题的分隔方式不一致

PubMed API 返回的标准 XML 具有固定结构，
解析逻辑简洁可靠，不需要大量正则匹配和边界情况处理。

## 七、常见问题

### Q1：双击 index.html 打不开？

**原因**：浏览器的 CORS 安全策略禁止 `fetch()` 加载本地文件。

**解决**：必须使用 `python main.py` 启动 HTTP 服务器访问。

### Q2：没有 Python 环境怎么办？

**解决**：使用任何静态文件服务器均可：

```bash
# Node.js
npx serve .

# PHP
php -S localhost:8080

# Python 内置（无需安装任何包）
python3 -m http.server 8080
```

### Q3：database.json 能否精简体积？

当前 2.5 MB（含换行缩进），可压缩至约 1.8 MB（去除缩进）。

如需进一步压缩，可删除低频字段（`pub_types`、`mesh_terms`），
或转为二进制格式（如 MessagePack），但需同步修改前端解析逻辑。

### Q4：能否部署到公网？

可以。将以下两个文件上传到任何静态托管服务即可：
- `index.html`
- `database.json`

推荐平台：GitHub Pages、Vercel、Netlify、Cloudflare Pages

### Q5：如何更新数据库？

如需新增文献：
1. 获取新文献的 PMID
2. 调用 PubMed API 获取 XML
3. 解析并追加到 `database.json`
4. 重新部署
